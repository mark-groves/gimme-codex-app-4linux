# Agent Notes

When making code, script, documentation, packaging, or workflow changes in this repository, first read:

`docs/working/project-discoveries-obstacles-lessons.md`

Use that file to avoid rediscovering prior decisions, especially around Electron packaging, Linux runtime behavior, native module rebuilds, Omarchy/Hyprland issues, and upstream appcast handling.

After a session that changes the repository or uncovers durable project knowledge, append a concise note to that document with any new discoveries, obstacles, or lessons.

For simple questions, status checks, or conversations that do not change the repository, do not update the project notes unless a genuinely durable project insight was discovered.
